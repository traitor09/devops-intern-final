# Grafana Loki Configuration Guide
## Step 1 - Deploy Loki and Promtail with Docker Compose  

Prepare a docker-compose.yml file to run Loki and Promtail.  
Although the file is not stored in this repository, its contents are shown below for reference.

```yaml
version: "3"

networks:
  loki:

services:
  loki:
    image: grafana/loki:2.9.2
    ports:
      - "3100:3100"
    command: -config.file=/etc/loki/local-config.yaml
    networks:
      - loki

  promtail:
    image: grafana/promtail:2.9.2
    volumes:
      - /var/log:/var/log
    command: -config.file=/etc/promtail/config.yml
    networks:
      - loki

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    networks:
      - loki
```

Run with:
```bash
docker-compose up -d
```
## Step 2 -Enable Log Shipping

To forward logs from Docker containers, configure Docker to use the Loki logging driver.
First, install the Loki Docker driver plugin:

```bash
docker plugin install grafana/loki-docker-driver:latest --alias loki --grant-all-permissions
```

Run your container with the Loki log driver:
```bash
docker run --log-driver=loki \
    --log-opt loki-url="http://localhost:3100/loki/api/v1/push" \
    --log-opt loki-batch-size=400 \
    devops-intern-hello
```

## Step 3- View Logs in Grafana

1. Open Grafana at `http://localhost:3000`.
2. Login (default admin/admin).
3. Add Data Source -> Loki -> URL `http://loki:3100`.
4. Go to Explore.
5. Select Loki datasource.
6. Query: `{container_name="devops-intern-hello"}`.
